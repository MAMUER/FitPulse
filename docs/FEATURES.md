|Вкладка|Содержимое|
|---|---|
|**Обзор**|Пульс, SpO₂, сон, давление, AI-анализ, тренировка на сегодня|
|**Профиль**|Возраст, вес, рост, цели, противопоказания, питание, сон|

## ML-классификация состояния

|Состояние|Описание|Рекомендации|
|---|---|---|
|**Восстановление**|Низкая нагрузка + высокий HRV + хорошее восстановление|Лёгкая активность, растяжка, йога|
|**Базовая выносливость E1-E2**|Работа ниже лактатного порога, устойчивая кардиореспираторная система|Бег/велосипед/плавание 45–90 мин|
|**Пороговая выносливость E3**|Нагрузка вблизи анаэробного порога, баланс лактата|Темповые интервалы 30–60 мин|
|**Силовая/HIIT**|Высокая вариабельность пульса + постнагрузочная гипертензия + стресс-реакция|HIIT, силовые, спринты|
|**Перетренированность**|Повышенный пульс в покое + низкий HRV + усталость + ухудшение сна|Отдых 1–3 дня, снизить объём|
|**Заболевание**|Повышение температуры + общая слабость + отклонения в детоксикации|Прекратить тренировки, обратиться к врачу|

**Источник:** `cmd/classifier/main.go` — правила-на-основе классификация по 7 биометрическим признакам.

## Генерация тренировок и диеты

Адаптивный план на основе:

- текущего состояния пользователя (классификация)
- цель
- место тренировки
- доступные дни и время суток
- подключённые устройства
- особенности здоровья пользователя, влияющие на результат

В том числе учитываются:

- аллергии и заболевания
- инвалидность и ограничения по подвижности
- женский цикл (ручной ввод; интеграция с Flo — endpoint готов, backend storage реализован, полная синхронизация pending)
- динамика веса и состав тела (ручной ввод; интеграция с OKOK International — endpoint готов, backend storage реализован, полная синхронизация pending)

**Тренировки:** 4–12 недель, упражнения с подходами/повторениями, расписание по дням и времени.

**Диета:** расчёт калорий по формуле Миффлина-Сан Жеора, типы диет (сбалансированная, высокобелковая, для похудения, кето, палео), макронутриенты, приёмы пищи по времени. Расчёт происходит на сервере (Python ML service).

## Административные возможности

- Регистрация через **invite-код** (одноразовый)
- Просмотр списка пользователей
- Создание новых invite-кодов
- Просмотр подключённых устройств пользователей

> Примечание: все админ-операции выполняются в `user-service` через gRPC. Gateway не имеет прямого доступа к БД.
